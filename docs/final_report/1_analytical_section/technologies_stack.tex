\section{Обзор программного инструментария}

Для реализации распределенной системы TechEventsHub был выбран современный технологический стек для обеспечения высокой производительности и отказоусточивости.

\subsection{Бэкенд}
В качестве основного языка программирования выбран \textbf{Python 3.11}, являющийся стандартом индустрии для задач парсинга и взаимодействия с моделями искусственного интеллекта.

В качестве HTTP-фреймворка выбран \textbf{FastAPI} благодаря встроенной поддержке асинхронности, автоматической генерации спецификаций OpenAPI и строгой валидации данных на базе Pydantic.

Для взаимодействия с Telegram Client API (MTProto) используется \textbf{Telethon}, поскольку данная библиотека поддерживает асинхронную работу и позволяет эффективно считывать историю сообщений из публичных каналов.

Для лексического анализа и очистки HTML-кода веб-страниц при парсинге классических сайтов применен \textbf{BeautifulSoup4}.

\subsection{База данных и кэширование}
В качестве основной реляционной СУБД выбрана \textbf{PostgreSQL}. 
Важным фактором при выборе стала поддержка типа данных JSONB, который позволяет эффективно хранить индивидуальные параметры фильтрации пользователей и выполнять агрегацию данных на уровне базы без дополнительных JOIN-запросов.

Для ORM-маппинга и версионирования схемы БД применены \textbf{SQLAlchemy} и \textbf{Liquibase}.

\textbf{Redis} выполняет функцию хранилища данных в оперативной памяти и применяется для кэширования результатов ресурсоемких запросов, а также хранения временных токенов для безопасного связывания учетных записей.

\subsection{LLM}
Для задачи суммаризации и извлечения параметров мероприятий из русскоязычных анонсов выбрана модель \textbf{YandexGPT 5.1 Pro}. 
Интеграция реализована через совместимую библиотеку OpenAI SDK, что обеспечивает стандартизированный подход к вызову API и позволяет в будущем легко заменить поставщика нейросети без переписывания бизнес-логики.

\subsection{Асинхронный транспорт}
В качестве отказоустойчивого брокера сообщений для передачи уведомлений между сревисом скрейпинга и Telegram-ботом внедрен \textbf{Apache Kafka}. 
Использование Kafka гарантирует доставку сообщений при пиковых нагрузках и позволяет маршрутизировать невалидные данные в отдельную очередь DLQ.

\subsection{Механизм оповещения пользователей}
Для отправки уведомлений на электронную почту и рендеринга HTML-шаблонов писем используются \textbf{aiosmtplib} и \textbf{Jinja2}.

\subsection{Клиентское приложение}
Пользовательский веб-интерфейс разработан на базе библиотеки \textbf{React} и фреймворка \textbf{Next.js} версии 14. 
Данный выбор обоснован поддержкой технологии серверных действий. 
Она позволяет инкапсулировать запросы к бэкенду, выполняя их на стороне промежуточного сервера. 
Такой подход предотвращает ошибки совместного использования ресурсов и обеспечивает безопасное хранение JWT-токенов авторизации с использованием файлов cookie, защищенных атрибутом HttpOnly.

Визуальное оформление интерфейса реализовано с применением библиотек \textbf{Tailwind CSS} и \textbf{Shadcn UI}.
Их использование способствует ускорению процесса разработки и гарантирует визуальное единообразие компонентов системы.

Проверка корректности вводимых пользователем данных (валидация сложных форм, включая диапазоны дат и выбор форматов) осуществляется с использованием библиотек \textbf{React Hook Form} и \textbf{Zod}. 
При этом клиентская схема проверки данных синхронизирована со контрактом OpenAPI.

\subsection{Инфраструктура и развертывание сервисов}
Для обеспечения идентичности сред разработки и продуктовой среды применена технология контейнеризации \textbf{Docker}. 
Автоматизированное тестирование бизнес-логики и процессов взаимодействия с базой данных реализовано посредством библиотеки \textbf{Testcontainers}.

Мониторинг состояния системы осуществляется на базе \textbf{Prometheus} и \textbf{Grafana}. 
Данное решение обеспечивает сбор бизнес-показателей, а также системных RED-метрик в режиме реального времени.
